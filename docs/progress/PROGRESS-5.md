## 要件サマリー

### 背景・目的
レビュー送信時、清潔度(星)と利用許可の **どちらが未入力かに関わらず** 常に同じメッセージ `errStarAccess`「星と『利用許可』を選んでください」が出ていた。問題は2点:
1. どちらのフィールドが欠けているか分からない（利用許可を選んでいても「選んでください」と出る）
2. 「星」という表記がフォーム上のラベル「清潔度」と不一致

### 受入条件
- [x] 1. 清潔度・利用許可ともに未選択 → `errBoth`「清潔度 と 利用許可 を選択してください」
- [x] 2. 清潔度のみ未選択 → `errStarOnly`「清潔度を選択してください」
- [x] 3. 利用許可のみ未選択 → `errAccessOnly`「利用許可を選択してください」
- [x] 4. i18n 4言語(ja/en/ko/zh)すべてに反映
- [x] 5. 旧キー `errStarAccess` は参照されなくなるため削除

### スコープ
- **対象**: `ReviewForm.tsx` の submit() バリデーション分岐 / `messages/{ja,en,ko,zh}.json` のキー差し替え
- **対象外**: report モード(notAToilet)の挙動変更 / API バリデーション / 他フォーム
